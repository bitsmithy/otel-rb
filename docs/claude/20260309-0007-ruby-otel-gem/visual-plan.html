Implementation Plan — Phase 2

otel-rb

A thin Ruby gem mirroring bitsmithy/go-otel — one call wires traces, metrics, and logs over OTLP/HTTP. Rack middleware for HTTP instrumentation. Logger formatter for trace-log correlation.

Telemetry::Setup.call(config). Wires TracerProvider + BatchSpanProcessor, MeterProvider + PeriodicMetricReader (rescue LoadError), LoggerProvider (rescue LoadError silent). Sets globals and composite W3C propagator. Returns { shutdown:, tracer:, meter: }.

Standard Rack middleware. initialize(app, tracer, meter). Pre-registers 3 metric instruments. Per-request: extract W3C context → start span → @app.call(env) → read route template → rename span → set attributes → record metrics.

Subclass of Logger::Formatter. Appends trace_id=<32hex> span_id=<16hex> suffix when OpenTelemetry::Trace.current_span.context.valid?. Zero call-site changes in the app.

Public API surface
Telemetry.setup(config = Config.new)
Primary entry point. Wires all three signals, sets globals, returns result hash. result = Telemetry.setup at_exit { result[:shutdown].call } tracer = result[:tracer] meter = result[:meter] # nil if metrics unavailable
Telemetry::Config.new(**opts)
All keyword args optional. Sensible defaults from runtime. Telemetry::Config.new( service_name: "my-app", service_namespace: "bitsmithy", service_version: ENV["GIT_COMMIT_SHA"], endpoint: ENV["OTEL_EXPORTER_OTLP_ENDPOINT"], log_level: :info )
Telemetry::Middleware
Standard Rack middleware. Pass tracer + meter from setup result. # config/application.rb config.middleware.use Telemetry::Middleware, result[:tracer], result[:meter]
Telemetry::TraceFormatter
Drop-in Logger formatter. Appends trace_id + span_id automatically. # config/initializers/telemetry.rb Rails.logger.formatter = Telemetry::TraceFormatter.new
Key design decisions
Decision Choice Rationale
Metrics unavailability rescue LoadError → warn + nil Idiomatic Ruby; works in all load paths; graceful degradation without crash
Logs unavailability rescue LoadError → nil (silent) Logs are truly optional; no warning needed; TraceFormatter covers the primary use case
Log correlation Logger::Formatter subclass Inherits msg2str + format_datetime; transparent (zero call-site changes)
TaggedLogging vs TraceFormatter TraceFormatter TaggedLogging requires .tagged { } at call site; TraceFormatter is automatic
Route label action_dispatch.route_uri_pattern Rails 7.1+ template (e.g. /users/:id) prevents high-cardinality metric labels
detach_context Not included (v1) Ruby OTel uses fiber-local context with no cancellation signal; Go pattern does not apply
Metrics as hard gemspec dep Yes (with LoadError guard) Best out-of-the-box experience; constrained envs can use Bundler without group
Future Rails/Rack integration: After the gem is built and published (or path-referenced), integration into any Rails app requires three files: add gem to Gemfile, create config/initializers/telemetry.rb calling Telemetry.setup and assigning Rails.logger.formatter, and insert config.middleware.use Telemetry::Middleware, ... into config/application.rb. No other application code changes required.
