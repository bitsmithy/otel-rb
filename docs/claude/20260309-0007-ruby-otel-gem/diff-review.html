api corrections discovered
Ruby OTel API corrections vs. plan
FIX-01
OTLP metrics require path. Plan specified require "opentelemetry/exporter/otlp/metrics" — incorrect. Actual path: require "opentelemetry/exporter/otlp_metrics". Class name OpenTelemetry::Exporter::OTLP::Metrics::MetricsExporter was correct.
FIX-02
TraceContext propagator. OpenTelemetry::Trace::Propagation::TraceContext is a module, not a class. Plan called .new — incorrect. Correct: TraceContext.text_map_propagator.
FIX-03
CompositeTextMapPropagator construction. Plan called .new(propagators: [...]) — .new is private. Correct factory: CompositeTextMapPropagator.compose_propagators([...]).
FIX-04
Rack header getter. Plan referenced OpenTelemetry::Common::HTTP::HeaderExtractor — does not exist. Correct: OpenTelemetry::Context::Propagation::RackEnvGetter.new.
FIX-05
MeterProvider#meter arity. Plan called mp.meter(name, version) — version is a keyword arg. Correct: mp.meter(name, version: version).
FIX-06
Rack::Test app override pattern. Tests that created local app = described_class.new(inner_xxx, ...) were still calling the outer let(:app) via Rack::Test's #app method. Fixed by using Rack::MockRequest.new(...).get(...) for those edge-case apps.
FIX-07
LoadError stub in setup_spec. Stubbing Kernel.require for a gem that's already loaded doesn't raise — require returns false for cached files. Fixed by stubbing MeterProvider.new to raise LoadError, which the rescue clause in setup_metrics catches.
NOTE-01
Ruby 4.0: logger not a default gem. Already addressed in the previous session: logger ~> 1.6 added to gemspec.
