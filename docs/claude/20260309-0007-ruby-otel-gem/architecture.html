Architecture — Phase 1 Research

otel-rb gem

A thin, opinionated OpenTelemetry setup library for Ruby/Rails services — the Ruby equivalent of bitsmithy/go-otel. One call wires traces, metrics, and logs over OTLP/HTTP.

Module Breakdown
lib/telemetry/setup.rb core

The main entry point. Telemetry.setup(config) creates and registers TracerProvider, MeterProvider, LoggerProvider. Sets the W3C TraceContext+Baggage composite propagator globally. Returns { shutdown:, tracer:, meter: }.

Telemetry.setup :shutdown :tracer :meter
lib/telemetry/config.rb core

Plain data object for configuration. All fields optional with sensible defaults: service_name from $PROGRAM_NAME, service_namespace from parent dir, service_version from gemspec, endpoint from OTEL_EXPORTER_OTLP_ENDPOINT env var.

service_name: service_namespace: service_version: endpoint: log_level:
lib/telemetry/middleware.rb http

Standard Rack middleware: initialize(app, tracer, meter). Fully composable — calls @app.call(env), never mutates env. Pre-registers metric instruments once on init. Extracts W3C trace context from headers; reads route pattern from env after routing for low-cardinality labels. Compatible alongside other OTel middleware.

.new(app, tracer, meter) #call(env)
lib/telemetry/trace_formatter.rb logging

A Logger::Formatter subclass that appends trace_id and span_id to every log message when an active OTel span is present in the current fiber-local context. Drop-in replacement for the Rails logger formatter.

#call(severity, time, progname, msg)
lib/telemetry.rb entry point

Top-level Telemetry module re-exporting the public API. No detach_context — Ruby OTel uses fiber-local context with no cancellation signal, so the Go pattern does not apply.

.setup ::Config ::Middleware ::TraceFormatter
spec/ tests

RSpec test suite using in-memory exporters and span recorders — no real collector needed. Tests cover: setup returns non-nil tracer/meter, endpoint override, middleware creates spans with correct attributes, metric instruments record correct values, formatter injects trace IDs.

setup_spec.rb middleware_spec.rb trace_formatter_spec.rb
go-otel → otel-rb Translation
Concept Go (go-otel) Ruby (otel-rb) Notes
Setup entry point telemetry.Setup(ctx, cfg) Telemetry.setup(config) No context arg — Ruby OTel uses fiber-local context
Config type telemetry.Config{...} struct Telemetry::Config.new(...) Keyword arguments, all optional
Return values shutdown, logger, tracer, meter, err { shutdown:, tracer:, meter: } Logger handled via formatter, not returned object
Shutdown shutdown func(context.Context) error shutdown = -> { ... } proc Call from at_exit or Rack close
HTTP middleware mw.Wrap(next) use Telemetry::Middleware, tracer, meter Standard Rack interface; app injected by Rack, fully composable
Log + trace correlation TraceHandler wraps slog.Handler Telemetry::TraceFormatter wraps Logger formatter Set on Rails.logger in initializer
Fanout logging FanoutHandler[]slog.Handler Rails broadcast / multiple outputs Rails already supports multi-device logging
Detached context telemetry.DetachedContext(ctx) Not included in v1 Ruby OTel fiber-local ctx has no cancellation signal; Go pattern does not apply
Route template r.Pattern (Go 1.22+ ServeMux) env["action_dispatch.route_uri_pattern"] Rails-specific; fallback to PATH_INFO for plain Rack
Service name default Last segment of module path File.basename($PROGRAM_NAME) Or from gemspec name field
Key Ruby-specific constraint: The Ruby OTel logs signal is still experimental in early 2026. opentelemetry-exporter-otlp-logs may have limited availability — logs integration should be optional/additive. Traces + metrics are stable and well-supported. The log formatter approach (injecting trace_id/span_id into Rails logger) provides the most pragmatic path to trace-log correlation without depending on experimental packages.
