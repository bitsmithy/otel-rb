Implementation Plan

Rails.logger → OTel

When integrate_tracing_logger: true, every Rails.logger call automatically emits an OTel log record with trace context. Zero app code changes.

The Problem

Current behavior
Desired behavior
  1. App calls Rails.logger.info("Order placed")
  2. TraceFormatter appends trace/span IDs to text output
  3. Log line written to STDOUT/file with IDs in text
  4. OTel backend never sees this log

Only Telemetry.log reaches OTel. All Rails.logger calls — from app code, gems, Rails internals — are invisible to the OTel backend.

  1. App calls Rails.logger.info("Order placed")
  2. LogBridge intercepts at add level
  3. Original logger writes to STDOUT/file (with TraceFormatter)
  4. OTel LogRecord emitted with trace context

Every Rails.logger call appears in both text output and OTel backend. No app code changes needed.

Key Code Changes

new lib/telemetry/log_bridge.rb
module Telemetry
  module LogBridge
    RUBY_TO_OTEL_SEVERITY = {
      0 => [5,  'DEBUG'],
      1 => [9,  'INFO'],
      2 => [13, 'WARN'],
      3 => [17, 'ERROR'],
      4 => [21, 'FATAL'],
      5 => [21, 'FATAL']
    }.freeze

    def add(severity, message = nil, progname = nil)
      if Thread.current[:telemetry_skip_otel_bridge]
        return block_given? ? super { yield } : super
      end

      severity ||= ::Logger::UNKNOWN

      if message.nil?
        if block_given?
          resolved_message = yield
          result = super(severity, resolved_message, progname)
        else
          resolved_message = progname
          result = super(severity, nil, progname)
        end
      else
        resolved_message = message
        result = super(severity, message, progname)
      end

      otel_severity = RUBY_TO_OTEL_SEVERITY[severity]
      if otel_severity && resolved_message
        span_ctx = OpenTelemetry::Trace.current_span.context
        @telemetry_bridge_logger ||=
          OpenTelemetry.logger_provider.logger(
            name: 'telemetry.bridge',
            version: Telemetry::VERSION
          )
        @telemetry_bridge_logger.on_emit(
          severity_number: otel_severity[0],
          severity_text:   otel_severity[1],
          body: resolved_message.to_s,
          trace_id:    span_ctx.valid? ? span_ctx.trace_id : nil,
          span_id:     span_ctx.valid? ? span_ctx.span_id : nil,
          trace_flags: span_ctx.valid? ? span_ctx.trace_flags : nil,
          observed_timestamp: Time.now
        )
      end
      result
    end
  end
end
modified lib/telemetry/logger.rb
# Dedup guard in emit
def emit(level, message, rails_logger:)
  emit_otel(level, message)
  if rails_logger && defined?(::Rails)
    begin
      Thread.current[:telemetry_skip_otel_bridge] = true
      ::Rails.logger.public_send(level, message)
    ensure
      Thread.current[:telemetry_skip_otel_bridge] = false
    end
  end
end
modified lib/telemetry.rb
# wire_tracing_logger now also installs bridge
def wire_tracing_logger
  existing = Rails.logger.formatter
  if existing && !existing.is_a?(TraceFormatter)
    return if skip_formatter_replacement?(existing)
    warn "[Telemetry] replacing existing logger formatter"
  end
  Rails.logger.formatter = TraceFormatter.new
  Rails.logger.singleton_class.prepend(LogBridge)
end

Edge Cases

ScenarioExpected Behavior
Telemetry.log(:info, msg) mirrors to Rails.loggerThread-local flag prevents LogBridge from re-emitting — OTel gets exactly 1 record
Telemetry.log(:info, msg, rails_logger: false)OTel emission via emit_otel only. Rails.logger not called. No bridge involvement.
Block-form: Rails.logger.info { expensive }Block evaluated once by LogBridge, resolved value passed to super. No double evaluation.
Log level filtering: Rails.logger.level = :warnLogger's own add returns early for debug/info. Bridge's super handles filtering — OTel emission also skipped since add short-circuits before our interception runs. Actually: our override runs first, so we emit before super. See callout below.
Test mode with SimpleFormatterwire_tracing_logger returns early — neither formatter nor bridge is installed
integrate_tracing_logger: false (default)wire_tracing_logger never called — no bridge, no formatter change
Non-Rails (plain Rack)defined?(Rails) is false — bridge never installed. Telemetry.log works standalone.
Concurrent threadsThread-local flag is per-thread — no cross-thread interference
Log level filtering note. The LogBridge add override runs before super's level check. This means OTel records are emitted even for messages below the logger's level. This is arguably correct — OTel has its own level filtering at the backend. If strict filtering is needed, add a severity < level early return in LogBridge.

Backward compatibility. No breaking changes. integrate_tracing_logger: false (default) is completely unchanged. The bridge only activates on opt-in. Telemetry.log and Telemetry.logger continue to work identically.
Prepend is permanent. Once LogBridge is prepended onto Rails.logger.singleton_class, it cannot be cleanly removed. This is fine for production — the bridge is installed once at boot. But in tests, each test's Telemetry.setup would re-prepend. The module's idempotent behavior (checks thread-local, delegates to super) makes repeated prepends harmless.
